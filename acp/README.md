# Architecture Change Proposals (ACP)

Každá architektonická změna musí projít tímto procesem.

**Pravidlo:** Implementace nikdy nepředbíhá schválení ACP.

## Konvence pojmenování

```
ACP-{NNNN}-{název}.md
```

Příklad: `ACP-0001-sharadar-provider.md`

## Lifecycle

```
DRAFT → PROPOSED → APPROVED | REJECTED | WITHDRAWN
```
